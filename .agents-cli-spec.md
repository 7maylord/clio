# Agent Spec: Ambient Expense Agent Prototype

## Overview
An ambient expense agent that automatically processes employee expense claims. It instantly approves claims under $100 and flags claims of $100 or more for manual review using a human-in-the-loop (HITL) pause.

## Example Use Cases
1. **Under $100 Claim:** 
   * Input: Expense of $45 for "Team Lunch".
   * Output: Claim is auto-approved.
2. **$100 or More Claim:**
   * Input: Expense of $150 for "Client Dinner".
   * Output: Execution is paused and prompts a human reviewer for approval. Once approved, the claim is approved.

## Workflow Structure (ADK 2.0)
* **Entry (START):** Receives the expense claim data (amount, description).
* **Router / Classifier:** Determines if the expense is under $100 or >= $100.
* **`auto_approve` Node:** Auto-approves expenses under $100.
* **`review_agent` Node:** Yields a `RequestInput` event to pause execution for human review for expenses >= $100.
* **Output:** The final approval status (Approved/Rejected) and details.

## Constraints & Safety Rules
* Any expense of $100 or more **must** go through human review.
* No automatic approval is allowed for claims of $100 or more.

## Success Criteria
* Correct routing based on the $100 threshold.
* Successful pause and resume sequence using `RequestInput` for expenses >= $100.
* Functional local prototype execution using `agents-cli run`.
